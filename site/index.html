Features

核心功能

從開店、上架、搜尋到下單出貨,覆蓋電商後台 SaaS 的完整流程。

Multi-tenancy

多租戶隔離

acts_as_tenant 行級隔離,每間店只看得到自己的資料;後台靠登入定租戶,店面靠網址 slug 定租戶。

Concurrency

防超賣

結帳用悲觀鎖(SELECT … FOR UPDATE)扣庫存,並發搶購不會扣成負數,附 race condition 測試。

State machine

訂單狀態機

AASM 管理 pending → paid → shipped,付款失敗整筆 rollback、不重複付款。

Background jobs

背景任務鏈

付款成功後觸發:通知店家 → 寄買家確認信 → 出貨準備的 job 鏈。

Search

商品搜尋

pg_search 以 Postgres trigram 搜尋,支援部分比對與中文,不另外架 Elasticsearch。

Storefront

公開店面 + 結帳

免登入店面,瀏覽/搜尋/下單;結帳串接綠界(ECPay)金流,付款後寄出確認信。

Checkout

結帳金流流程

串接綠界(ECPay):導頁付款 + webhook 驗章,跟業界 hosted 金流同一套。

01

建立訂單

結帳時先建一筆 pending 訂單與交易編號。

02

導去綠界

帶簽章參數導到綠界付款頁,買家在綠界完成付款。

03

驗章 webhook

綠界 server 回呼,後端重算 CheckMacValue 比對,確認沒被竄改。

04

標記付款

驗章通過才轉為 paid 並觸發通知;重複回呼不重複處理。

Decisions

技術決策

依專案規模選技術,不過度設計。

Multi-tenancy
acts_as_tenant 行級隔離
不用 Apartment schema 隔離:商家一多,每張表都要逐 schema 跑 migration。
Search
pg_search trigram
不用 Elasticsearch:這規模不需要多養一個叢集、吃記憶體。
Orders
AASM 狀態機
狀態與轉移顯式、好讀好測,勝過土砲 enum + if。
Concurrency
悲觀鎖防超賣
高併發搶購最穩;樂觀鎖會大量 retry。
Frontend
Hotwire + REST
單一技術棧、開發快;不為用不到的互動多養一套 SPA。
