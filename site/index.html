MerchantOS

多租戶電商 SaaS——商家註冊後開店、管理商品與訂單,散客可在公開店面瀏覽、搜尋、結帳。類似輕量版 Shopify。

核心功能

每個功能都對應一個常見的 Rails 面試考點。

多租戶隔離

acts_as_tenant 行級隔離,每間店只看得到自己的資料;後台靠登入定租戶,店面靠網址 slug 定租戶。

防超賣

結帳用悲觀鎖(SELECT … FOR UPDATE)扣庫存,並發搶購不會扣成負數,附 race condition 測試。

訂單狀態機

AASM 管理 pending → paid → shipped,付款失敗整筆 rollback、不重複付款。

背景任務鏈

付款成功後觸發通知店家 → 寄買家確認信 → 出貨準備的 job 鏈。

商品搜尋

pg_search 以 Postgres trigram 做搜尋,支援部分比對與中文,不另外架 Elasticsearch。

公開店面 + 結帳

免登入店面,瀏覽/搜尋/下單;結帳填 email 收確認信(金流目前為 mock,可接綠界)。

技術決策(為何這樣選)

依專案規模選技術,不過度設計。

面向選用為何不選其他
多租戶acts_as_tenant 行級不用 Apartment schema 隔離:商家一多,每張表都要逐 schema 跑 migration
搜尋pg_search trigram不用 Elasticsearch:這規模不需多養一個叢集
訂單AASM 狀態機狀態與轉移顯式、好測,勝過土砲 enum + if
防超賣悲觀鎖高併發最穩;樂觀鎖會大量 retry
API/前端Hotwire + REST單一技術棧、開發快,不為用不到的互動多養 SPA

工程實踐

測試

RSpec + FactoryBot,每個功能都附測試,含並發防超賣的 race condition spec。

CI

GitHub Actions:RuboCop、RSpec、brakeman 安全掃描、gitleaks、Docker build。

觀測

Sentry 錯誤追蹤、Lograge 單行 JSON log,每行帶 tenant_id / user_id。
